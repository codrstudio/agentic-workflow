# Verify Web Build

Valide o build do web app e certifique-se de que todas as páginas foram migradas com sucesso.

## Inputs

1. Leia `{sprint}/features.json` — todas as features esperadas
2. Verifique o estado de cada feature (passing vs failing/pending)
3. Inspecione `{web_dir}/routes/web/` — rotas criadas
4. Verifique `{web_dir}/src/routeTree.gen.ts` ou gere se necessário

## Procedimento

1. **Validação de TypeScript**:
   - Rode `npx tsc --noEmit` em `{web_dir}/`
   - Relat todos os erros encontrados

2. **Validação de Build**:
   - Rode `npx vite build` em `{web_dir}/`
   - Se falhar, liste erros
   - Se sucesso, anote hash do build

3. **RouteTree**:
   - Verifique se `routeTree.gen.ts` existe e é recente
   - Se não existir, tente gerar manualmente (pode ser necessário command de TanStack Router)
   - Certifique-se de que todas as rotas estão registradas

4. **Feature Completude**:
   - Leia features.json
   - Conte features com status `passing`
   - Calcule taxa de completude (passing / total)
   - Identifique features `pending`, `failing`, ou `blocked`

5. **Navegação**:
   - Verifique que ProjectNav tem todas as abas principais
   - Verifique que DynamicBottomNav tem shortcuts configuradas
   - Verifique AppSidebar integrado

## Success Criteria

✅ **Sucesso**: Todas as features com status `passing`, build sem erros, todas as rotas registradas.

⚠️ **Parcial**: Maioria completa, poucas features pendentes ou bloqueadas (pode chainear para próxima wave).

❌ **Falha**: Build quebrado, muitas features failing, ou problemas críticos na navegação.
